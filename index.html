模型输出格式

每个模型输出 → { direction: UP/DOWN, confidence: 0-100%, expected_return: ±%, target_price: $ }

分类器预测方向(UP/DOWN)，回归器预测收益率(%)，两者结合得到置信度和目标价格。

Step 1: Ensemble加权投票

1
收集投票

11个模型各自给出方向(UP/DOWN)和置信度(0-100%)

2
加权累计
votes_up += confidence / 100 (如果方向=UP) votes_down += confidence / 100 (如果方向=DOWN)
3
计算共识
方向 = UP if votes_up > votes_down else DOWN 看涨概率 = votes_up / (votes_up + votes_down) × 100% 置信度 = min(100, |看涨概率 - 50| × 2 + 50)
4
预期收益
expected_return = clip(mean(各模型收益率), -10%, +10%)

Step 2: Polymarket信号调整

A
解析市场

从Polymarket获取BTC相关市场，提取价格目标、YES概率、交易量、到期日

B
计算权重
时间权重 = min(1.0, 30天/距到期天数) — 越近期越重要 资金量权重 = log₁₀(交易量) - 4 — 资金越多越可信 综合权重 = 时间权重 × 资金量权重
C
看涨评分
如果目标价 > 当前价: bullish += 权重 × YES概率 如果目标价 < 当前价: bullish -= 权重 × (1 - YES概率) 看涨分数 = 50 + (加权看涨 / 总权重 × 100) — 范围0-100
D
调整预测
调整幅度 = clip((看涨分数 - 50) / 500, -5%, +5%) 最终收益率 = 模型收益率 + 调整幅度 × min(预测小时/24, 3.0) 最终价格 = 当前价 × (1 + 最终收益率)

示例：看涨分数75 → 调整=(75-50)/500=+5% → 24H预测上调5%

评估指标

指标计算方式含义
准确率 正确预测数 / 总预测数 方向预测的对错比例，>50%即有预测能力

衍生品指标

指标全称含义信号解读
资金费率 (FR) Funding Rate 永续合约多头定期支付给空头的费用（或反向） 正费率=多头付费=市场偏多，极高费率(>0.05%)=可能回调
多空比 (L/S) Long/Short Ratio 持有多头仓位的账户数 / 空头账户数 >1=多头主导，>2=极端偏多（反向指标，可能见顶）
买卖比 Taker Buy/Sell Ratio 主动买入成交量 / 主动卖出成交量 >1=买方强势（主动吃卖单），<1=卖方强势
持仓量 (OI) Open Interest 所有未平仓合约的总价值 上升=新资金流入，下降=资金撤出。配合价格上涨+OI上升=健康趋势

订单簿指标

指标含义信号解读
价差 (Spread) 最优买价和最优卖价之间的差距 小=流动性好，大=流动性差或剧烈波动
失衡度 (Imbalance) (买单量-卖单量) / (买单量+卖单量) >0.1=买盘主导（短期看涨），<-0.1=卖盘主导

期权指标

指标全称含义信号解读
IV Implied Volatility 期权市场对BTC未来波动率的定价 高IV=预期大波动(不确定方向)，低IV=预期平稳
PCR Put/Call Ratio 看跌期权持仓量 / 看涨期权持仓量 >0.7=偏空(更多人买保险)，<0.5=偏多
MaxPain 最大痛点 让最多期权买家亏损的价格点 到期日价格倾向靠拢MaxPain（做市商操控理论）

链上指标

指标含义信号解读
HR (算力) 全网比特币挖矿算力 (EH/s) 持续上升=矿工看好未来收益=长期看涨，骤降=矿工关机/监管风险
Mempool 待确认交易数量和大小 拥堵=链上活跃度高=需求旺盛，空池=使用率低
Fee (手续费) 最快确认所需的手续费率 (sat/vB) 高=网络繁忙(可能是大行情)，低=网络空闲
